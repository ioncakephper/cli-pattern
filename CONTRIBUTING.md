# Contributing

We welcome contributions to this project! Please follow these guidelines to ensure a smooth development process.

## Pull Requests

1.  Fork the repository and create your branch from `main`.
2.  If you've added code that should be tested, add tests.
3.  Ensure the test suite passes.
4.  Make sure your code lints.
5.  Issue that pull request!

## Issues

We use GitHub issues to track public bugs. Please ensure your description is clear and has sufficient instructions to be able to reproduce the issue.

## Code of Conduct

By participating in this project, you are expected to uphold our [Code of Conduct](RULES_OF_CONDUCT.md).
